There's a point in every legacy project where you realize the real problem isn't the code — it's the absence of discipline around how you touch it. I hit that point staring at 367 lines of Flask with SQL injection, MD5 passwords, a global DB connection, and an admin route with literally zero auth. The comment at line 2 said it all: "do not touch without approval from Jean-Michel." Jean-Michel had left the building.

The situation: a REPL without a runtime

My first instinct was to open Claude Code and start prompting. That's the trap. A REPL without a runtime is just a chat with extra steps. No memory across sessions. No gate to stop you from skipping phases. No audit trail. You end up with hallucinated progress — the illusion of refactoring without the discipline of it.

I needed something that would force the work to stay honest.

The task: build the discipline layer first

Before touching a single line of the legacy code, I built the orchestration system:

  • A SQLite state machine — 7 tables tracking phases, smells, decisions, metrics, session continuity.
  • 9 slash commands — /survey, /map, /audit, /draft, /stabilize, /derive, /extend, /compact, /doctor.
  • Phase gates — check_gate.py validates exit criteria before any phase transition. No gate, no move.
  • Context injection — session/context.yaml auto-generated from SQLite, ~422 tokens. Claude reads state, not conversation history.
  • Feedback loops — a metrics dashboard with sparklines, regression detection, inter-session trend tracking.

The rule was simple: never modify src/ before /stabilize completes with all tests green.

The action: make the system visible

Running make verify before every session — 33 checks, 0 failures — changed how I felt about opening the project. Not anxious. Ready.

Running make smells and seeing the 5 critical smells unchanged was a reminder that the legacy wasn't going away on its own. The system kept score so I didn't have to carry it mentally.

The first real run of /survey through Claude Code catalogued all 5 smells into the DB, wrote logs/audit/survey.md, validated the gate. Then make dump committed the state to git — human-readable SQL, not a binary blob.

The result: discipline as infrastructure

The refactor hasn't started yet. That's the point. What exists now is the infrastructure that makes the refactor trustworthy: a pre-flight check, a state machine, phase gates, a feedback loop, and a context injection system that survives session boundaries without burning tokens on conversation history.

When /derive runs, it will have coverage metrics to defend against. When /stabilize runs, it will need green tests before the gate opens. Jean-Michel's code will be touched — but under controlled conditions.

The lesson: I evolve therefore I am

I used to think the hardest part of legacy refactoring was the code. It's not. It's the cognitive load of not knowing what you've already touched, what broke, and what was intentional.

Building the runtime first isn't overhead — it's the work. The code will follow. The discipline had to come first.
